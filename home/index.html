Snaption more_vert

Snaption is a social Android application for captioning and uploading images with your friends!

Snaption close Sept 2016 - Present

Snaption is a card game produced by Pic me! Games, Inc. For my Senior Capstone project, our goal was to release a mobile version of the game as a native Android application.

The application's backend is a Django Python server hosted on Google Cloud Platform's Compute Engine. The server also interacts with Google's Cloud SQL and Storage for database storage and image storage.

The Android application itself uses the Android Support Library for Material components, Glide for image loading and caching, RxJava 2 for asynchronous network requests and thread switching, and Retrofit for a REST API interface. The goal of the application was also to follow a Model-View-Presenter clean architecture for testability.

Through the application, users can log in through either Facebook or Google. Users can view photos uploaded by other users on a staggered grid, caption them, and upload a photo of their own.

Cru Central Coast more_vert

Cru Central Coast is an Android application for ride-sharing, events, and resources.

Cru Central Coast close Jan 2017 - Present

Campus Crusade for Christ Central Coast is a Christian Organization for college students along the central coast of California. For my Senior Project, my teammate and I worked to update and refresh the Android application for release.

The application's backend is a Node.js server using Keystone.js hosted on Heroku. The server also interacts with an S3 AWS instance for image storage, and MLabs for a MongoDB database.

The Android application itself uses the Android Support Library for Material components, Picasso for image loading and caching, RxJava for asynchronous network requests and thread switching, and Retrofit for a REST API interface.

Through the application, users can view upcoming events, request and provide rides, and view resources, and YouTube videos. There is also support for Leader log-in with administrative functionality.

Pokébase more_vert

Pokébase is an Android application for keeping track of the first 721 Pokémon, items, and more.

Pokébase close May 2016 - Feb 2017

Gotta catch 'em all! Pokébase for Android is an application that allows you to see information about the first 721 Pokémon. I developed this application for an introductory database class along with fellow classmate Brittany Berlanga.

The initial version of the application was backed by a Google Cloud App Engine using Cloud Endpoints written in Java and a Google Cloud SQL instance for storing information about Pokémon. The Google provided APIs allowed us to call server methods to access the database directly from the application.

Once the free trial for Google Cloud Platform expired, I migrated the application over to using a local SQLite database in order to preserve the functionality of the application. This required copying over the database file from the .apk file to the phone's storage upon launching the application for the first time.

The Android application itself uses the Android Support Library for Material components and Glide for image loading and caching.

Users can browse information about the first 721 Pokémon, various moves, items, and types. They can also create teams of up to six Pokémon, customizing each Pokémon's name, leve, and moveset.
